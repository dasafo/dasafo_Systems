# 🗂️ SKILL CATALOG | Dasafo Factory v5.0-MCP

Este catálogo describe no solo la función de cada herramienta, sino su papel en el flujo industrial de la **Factoría Dasafo v5.0-MCP**:

* **Fase de Descubrimiento y Estrategia (M1):**
  * `Apify Trend Analysis`: Utiliza actores de Apify para extraer inteligencia competitiva en tiempo real sobre nichos de mercado. Esencial para fundamentar decisiones de producto en datos reales antes de escribir una sola línea de código.
  * `Project Management`: El cerebro administrativo. Registra hitos, genera reportes de progreso y asegura que el estado del proyecto en disco refleje la realidad de la ejecución, manteniendo el "Vibe" del equipo.
  * `PRP Generator`: Crea el contrato industrial PRP_MASTER y las especificaciones SPEC_LITE. Define el "qué" y el "cómo" atómico de cada tarea, garantizando que el desarrollo sea predecible y medible.
  * `Social Content Strategy`: Convierte el contenido pilar (como este código) en activos sociales optimizados para diversas plataformas (Twitter, LinkedIn, Reels), manteniendo la voz de marca y la eficiencia de recursos.
  * `Startup Metrics Framework`: Traduce la ejecución técnica (segundos, bytes) en KPIs financieros de negocio SaaS como CAC, LTV y ROI. Fundamental para validar la viabilidad económica del proyecto.

* **Fase de Arquitectura (M2):**
  * `API Contract Generator`: Implementa un enfoque "Design-First" generando contratos OpenAPI 3.1. Establece la interfaz de comunicación antes de la implementación, permitiendo el desarrollo paralelo de frontend y backend.
  * `Architecture Decision Records (ADR)`: Gestiona el registro inmutable de por qué se toman decisiones técnicas críticas. Previene la "amnesia arquitectónica" y facilita el onboarding de futuros agentes o humanos.
  * `ArXiv Technical Digest`: Realiza investigaciones de vanguardia en ArXiv para asegurar que las soluciones propuestas aprovechen los últimos avances en IA y computación, elevando el nivel técnico del proyecto.
  * `Database Architect Strategic`: Diseña modelos de datos estratégicos con un enfoque en la escalabilidad y eficiencia de consultas, asegurando que la estructura de la base de datos soporte el ciclo de vida del producto.
  * `Research Manager`: Organiza y persiste investigaciones técnicas y arquitectónicas en la jerarquía de documentación del proyecto, creando una base de conocimiento sólida y accesible.

* **Fase de Producción (M3):**
  * `Async FastAPI Logic`: Scaffolding automático de backend con FastAPI siguiendo patrones DDD (Domain Driven Design). Implementa repositorios, servicios y modelos asíncronos para alto rendimiento.
  * `Atomic Design Tokens`: Gestiona y sincroniza tokens de diseño (colores, tipografías, espaciado) en JSON/CSS. Garantiza la coherencia visual total entre diseñadores y desarrolladores frontend.
  * `Frontend UI Designer`: Valida y genera sistemas de diseño basados en Tailwind y Shadcn. Asegura que la interfaz de usuario sea no solo funcional, sino visualmente premium y resiliente.
  * `Node.js Backend Patterns`: Aplica patrones industriales como Repositorio, DTO y Capa de Servicio en proyectos Node.js, garantizando una base de código limpia y fácil de mantener.
  * `Shadcn Component Library`: Administra la biblioteca de componentes de UI siguiendo principios de diseño atómico. Facilita la creación de interfaces de usuario complejas y modulares.
  * `Supabase Stack Expert`: Ingeniería de base de datos específica para Supabase/Postgres. Optimiza el rendimiento, la seguridad (Row Level Security) y el diseño del esquema.

* **Fase de Calidad y Seguridad (M4):**
  * `Agentic Thought Secret Scanner`: Un escáner de seguridad Zero-Trust diseñado para detectar secretos, claves API y credenciales expuestas en la base de código antes de cualquier commit.
  * `API Docs Generator`: Transforma los contratos OpenAPI en documentación Markdown profesional y legible para humanos, asegurando que la API sea fácil de consumir y entender.
  * `Factory Audit Pro`: Realiza auditorías de calidad de grado industrial, buscando anti-patrones y asegurando el cumplimiento de los estándares de la factoría.
  * `Playwright E2E Tester`: Ejecuta pruebas de extremo a extremo automatizadas en navegadores reales, capturando trazas e informes de errores para garantizar la fiabilidad del usuario final.
  * `Pytest Logic Verifier`: Genera y ejecuta pruebas unitarias y de integración en Python basadas en los criterios de las Spec-Lite, verificando que el código cumpla con el contrato de negocio.

* **Fase de Evolución e Infraestructura (M5):**
  * `Context Pruning Sieve`: Un filtro avanzado para optimizar el uso de tokens en los prompts, eliminando información irrelevante y enfocando la capacidad de razonamiento de la IA.
  * `Deployment Health Check`: Monitoriza en tiempo real la salud de los servicios desplegados y activa protocolos de auto-sanado (Auto-Healing) si detecta fallos en los endpoints.
  * `Docker Stack Provisioner`: Crea y gestiona la infraestructura como código (IaC) mediante Dockerfiles y Docker Compose, asegurando la reproducibilidad total del entorno.
  * `Skill Refactor Pro`: Aplica refactorizaciones quirúrgicas en el código siguiendo reglas de SI (Structural Integrity) para eliminar "smells" técnicos y mejorar la salud del repositorio.
  * `Terraform IaC Builder`: Aprovisiona infraestructura en la nube siguiendo los estándares de HashiCorp, permitiendo un despliegue escalable y seguro.

* **Herramientas Core:**
  * `Delegate Clean Session`: El núcleo de la multi-agencia. Delega tareas a sub-agentes en sesiones aisladas, utilizando Neo4j para inyectar "Golden Rules" y asegurar la calidad del output.
  * `Factory Doctor`: El médico general. Diagnostica la salud estructural del repositorio y verifica las dependencias, asegurando que la factoría funcione sin fricciones.
  * `Hallucination Guardrail`: Una capa de seguridad que previene alucinaciones del modelo mediante validaciones factuales, garantizando la fiabilidad técnica del sistema.
  * `Kanban Solidity Gate`: Implementa la regla de "Cero Pendientes" y proporciona un dashboard visual del estado de las tareas, manteniendo el flujo de trabajo ágil y transparente.
  * `Project Backbone Validator`: Verifica que el "esqueleto" físico del proyecto exista y cumpla con la estructura de carpetas requerida antes de proceder con el desarrollo.
  * `Registry Manager`: Administra el registro central de proyectos y variables de estado, manteniendo la verdad única sobre qué proyectos están activos y en qué fase se encuentran.
