## 🕹️ Operational Rules

1. **Stark-Solidity Guard (v3.3.0-S):** All skills reside in `06_SKILL_LIBRARY/` and are invoked via `skill_engine.py`. **NO MODIFICATION** of skills without explicit Human-in-the-Loop signature.
2. **SSoT:** The only sources of truth are the Global Pillars and the physical JSON artifacts (`PROJECT_STATE.json` and `TASKS/`) of each project.
3. **Zero-Trust Synchronization:** Progression through M1-M5 phases requires strict physical synchronization of task artifacts. No phase hopping.
4. **Traceability:** Every error must be distilled into the project's `FEEDBACK-LOG.md` and strictly parsed into JSON schema for AutoShield.
