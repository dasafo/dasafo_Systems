# 📊 EVALUATION_METRICS

## 1. Technical Health
- **Solidity Index**: Ratio of code passing security audit / total code.

## 2. Operational Efficiency
- **Token Density**: (Output Utility) / (Context Length). Lower is better.
- **Inference Latency**: Target <5s for critical decision gates.
- **DAG Depth**: Number of steps required to complete a project milestone.

## 3. Solidity & Integrity
- **Audit Signature Presence:** Binary (1 or 0). No transition to M5 without 1.
- **Segregation of Duties Adherence:** Detected self-validations trigger a hard rollback.
- **Unit Test Coverage:** Minimum 80% for critical logic.
- **Visual Vibe**: Quality of UI (evaluated via Playwright visual diffs).

## 4. Learning Velocity (AutoShield)
- **MTTD (Mean Time to Distill)**: Time from error occurrence to YAML feedback entry.
- **Recursion Prevention Rate**: Percentage of errors that never occur again after distillation.

## ⚖️ 05. Evaluation Metrics
>
> **Standard:** v3.1.5 "Solidity Guard"

- **SI Compliance**: 100% enforcement of International System units.
- **PRP Sync**: Alignment between initial `PRP_CONTRACT.json` and final `walkthrough.md`.
