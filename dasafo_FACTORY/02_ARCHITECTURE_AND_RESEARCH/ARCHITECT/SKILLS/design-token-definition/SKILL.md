# 🎨 Skill | Design Token Definition
> **Version:** v3.1.5 "Solidity Guard"
> **Source:** https://skills.sh/wshobson/agents/design-system-patterns
> **Agent:** ARCHITECT

## Objective
Define the visual DNA of the project (Tokens) to ensure the "Dasafo Vibe" (Glassmorphism, High-End Aesthetic) is consistent.

## Token Classes
1.  **Colors:** `brand-electric-blue`, `anomally-neon-red`, `bg-glass-dark`.
2.  **Spacing:** Semantic grid (8px base).
3.  **Typography:** Modern fonts (Outfit, Inter).
4.  **Effects:** Blur values for glassmorphism, transition timings.

## Workflow
1.  **Scan Standards:** Read `01_CODING_STANDARDS.md` for vibe rules.
2.  **Produce Tokens:** Generate a `design_tokens.json` in `$TARGET_PROJECT/LOCAL_KNOWLEDGE/architecture/`.
3.  **Instruct:** Create the `Visual_Standard.md` to be used by the `FRONTEND_DEV`.

## Constraint
Reject any "plain" or "browser-default" styles. Aesthetics must be PREMIUM.
