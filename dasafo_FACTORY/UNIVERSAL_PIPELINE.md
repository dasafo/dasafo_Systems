## 🛣️ The Journey


### 🔍 Phase M1: Discovery

- **Owner:** `PRODUCT_OWNER`
- **Output:** Draft `PRP_CONTRACT.json`.
- **Validation:** User approval of the vision and scope.


### 📐 Phase M2: Architecture

- **Owner:** `ARCHITECT`
- **Output:** System design and `API_CONTRACTS.md`.
- **Validation:** Signed `PRP_CONTRACT.json` (The Gate).


### ⚙️ Phase M3: Production

- **Owner:** `BACKEND_DEV` / `FRONTEND_DEV`
- **Output:** Functional, project-agnostic code.
- **Validation:** Unit tests and SoC compliance.


### 🧪 Phase M4: Compliance

- **Owner:** `QA_TESTER` / `DOCS_MASTER`
- **Output:** Security audit and technical walkthrough.
- **Validation:** Zero-Trust verification and documentation complete.


### 🔄 Phase M5: Operations

- **Owner:** `DEVOPS_SRE` / `FACTORY_EVOLVER`
- **Output:** Production deployment and memory distillation.
- **Validation:** System stability and semantic index update.
- **DEPLOY_OFFER:** Agents must proactively provide launch commands and ask: *"¿Deseas que levante los servicios ahora mismo?"*
